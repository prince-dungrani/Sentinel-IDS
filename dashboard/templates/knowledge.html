WHAT IS AN IDS?
Intrusion Detection System (IDS)

An IDS is a security tool that monitors network traffic or system activity for malicious behavior or policy violations. When it detects suspicious activity, it generates an alert for a SOC analyst to investigate.

Unlike a firewall (which blocks traffic), an IDS is passive — it observes and reports without actively blocking.

NIDS vs HIDS
NETWORK IDS (NIDS)
Monitors all traffic passing through a network segment. Sentinel-IDS is a NIDS.
HOST IDS (HIDS)
Monitors individual host system calls, logs, and file changes.
DETECTION METHODS: SIGNATURE vs HEURISTIC vs ML
🔑 Signature Engine

Matches packet payload against a database of known attack patterns (like antivirus signatures). Fast and precise but cannot detect unknown (zero-day) threats.

Example: Detect "union select" in HTTP payload → SQL Injection alert
📊 Heuristic Engine

Detects behavioral anomalies based on thresholds and rules (e.g., too many SYN packets). Can catch unknown attacks but may have more false positives.

Example: 50 SYN packets in 10s from one IP → SYN Flood alert
🤖 ML Engine

Uses trained models to predict threats from packet features. Combines supervised (RandomForest) and unsupervised (IsolationForest) learning for adaptive detection.

Example: Feature vector scores 0.85 → High-confidence anomaly alert
